# Contribuir

¡Gracias por tu interés en contribuir a Social Bot Scheduler! A continuación se indican las pautas para participar.

## Cómo contribuir

1. Abre un *issue* describiendo la propuesta o el bug.
2. Para cambios significativos, discute primero la implementación en un *issue*.
3. Crea una rama con un nombre claro: `feature/xyz` o `fix/descr`.
4. Envía un *pull request* con una descripción clara y los pasos para probar los cambios.

## Estilo de código
- Sigue la convención de Python (PEP8) para `bot.py` y otros scripts.
- Incluye tests cuando sea posible.

## Tests y CI
- Asegúrate de que los tests pasen antes de enviar un PR.
- Describe cómo ejecutar pruebas en la descripción del PR.

## Revisión y aprobación
- Los PR serán revisados por los mantenedores y pueden requerir cambios antes de ser fusionados.

## Código de Conducta
Se espera que todas las interacciones sean respetuosas y profesionales. Por favor, respeta el **Código de Conducta**: https://www.contributor-covenant.org/es/version/2/0/code_of_conduct/

Resumen: evita lenguaje discriminatorio, respeta diversidad y actúa de forma colaborativa.

---

Si necesitas ayuda para comenzar, abre un *issue* y te orientamos sobre tareas adecuadas para nuevos contribuyentes.
